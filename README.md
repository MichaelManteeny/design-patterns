# Паттерны проектирования для стажёров

Интерактивный учебный сайт по классическим паттернам проектирования «банды четырёх» (GoF). Материал рассчитан на стажёров без опыта в GoF и сопровождается живыми визуализациями, которые показывают, как каждый паттерн работает изнутри.

## 🎯 Зачем этот сайт

- **Аудитория** — стажёры, которые впервые сталкиваются с порождающими, структурными и поведенческими паттернами и хотят понять не только «что это», но и «когда применять».
- **После прохождения** вы научитесь распознавать типовые проблемы в коде, выбирать подходящий паттерн и читать диаграммы классов из реальных проектов.

## 📚 Какие паттерны

На сайте собраны десять ключевых паттернов, разделённых на три классические группы.

### Порождающие (Creational) — отвечают за создание объектов
- [Factory Method](src/content/patterns/factory.md) — делегирует создание объектов подклассам.
- [Builder](src/content/patterns/builder.md) — пошаговая сборка сложных объектов.
- [Singleton](src/content/patterns/singleton.md) — единственный экземпляр класса.

### Структурные (Structural) — отвечают за композицию классов и объектов
- [Adapter](src/content/patterns/adapter.md) — совмещает несовместимые интерфейсы.
- [Decorator](src/content/patterns/decorator.md) — динамически добавляет поведение.
- [Composite](src/content/patterns/composite.md) — единообразно работает с деревьями объектов.

### Поведенческие (Behavioral) — отвечают за взаимодействие объектов
- [Observer](src/content/patterns/observer.md) — оповещает подписчиков о событиях.
- [Strategy](src/content/patterns/strategy.md) — подменяет алгоритм на лету.
- [State](src/content/patterns/state.md) — меняет поведение при смене состояния.
- [Command](src/content/patterns/command.md) — превращает действие в объект.

## 🛠 Стек

- **Astro 5+** — статический сайт-генератор с островной архитектурой.
- **React** (через `@astrojs/react`) — для интерактивных визуализаций.
- **Vanilla Extract** — типобезопасные стили с нулевым рантаймом.
- **TypeScript** — строгая типизация всего проекта.
- **pnpm** — пакетный менеджер для быстрой установки зависимостей.

## 🚀 Локальный запуск

```bash
pnpm install
pnpm dev      # dev-сервер на http://localhost:4321/design-patterns
```

## 📦 Сборка

```bash
pnpm run build
pnpm run preview
```

Готовая статика окажется в каталоге `dist/` и готова к деплою на любой статический хостинг.

## 🌐 Деплой

- Автоматический деплой через **GitHub Actions** при пуше в `main`.
- Конфигурация — в `.github/workflows/deploy.yml`.
- Хостинг — **GitHub Pages**.
- Адрес сайта — [https://MichaelManteeny.github.io/design-patterns](https://MichaelManteeny.github.io/design-patterns).

## 🎨 Дизайн

- Тёмная тема включена по умолчанию, светлая переключается кнопкой в `header`.
- Оранжевая палитра акцентов с базовым цветом `#ff6b35`.
- Для каждого паттерна сделана интерактивная React-визуализация — её можно покрутить прямо на странице.
- Выбранная тема сохраняется в `localStorage` и подхватывается при следующем визите.

## 📁 Структура проекта
